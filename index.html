Configuration API

Customize everything through the init object. Here is the complete payload. 

new window.TechCard({
  // 1. Global Options
  card: "solo", // "solo" | "solo-mini"
  theme: "dark", // "dark" | "light"
  creditsText: "Developed by",

  // 2. Profile Details
  user: {
    name: "Kouya Tosten",
    role: "Full-Stack Dev | AI & Auto",
    bio: "Full-stack dev exploring Laravel/React.",
    avatar: "https://example.com/me.jpg",
    qrcode: "/imgs/qrcode.png",
    social_long: false,
    socials: [
      { name: "GitHub", url: "https://github.com/Tostenn" },
      { name: "LinkedIn", url: "https://linkedin.com" }
    ]
  },

  // 3. Expertise & Stats
  content: {
    techStack: {
      backend: [
        { name: "Laravel", version: "9.0" }
      ],
      frontend: [
        { name: "React", version: "18.0" }
      ]
    },
    projectGoal: "## Goal\nExplain your vision in markdown.",
    stats: [
      {
        label: "visitors",
        value: 1000,
        description: "Total active visitors"
      }
    ]
  }
}).open();

1. Global Options

  • card : Display format. Use "solo" for full profile, "solo-mini" for compact.
  • theme : UI colors. "dark" (default) or "light".
  • creditsText : Text displayed at the bottom of your card.

2. Profile (user)

  • name / role / bio : Standard text fields.
  • avatar : URL to your picture.
  • qrcode : Link to a generated QR Code image (e.g. your vCard or website).
  • socials : Array of { name, url }. Icons are auto-mapped based on the name string.
  • social_long : Set to true to display large social buttons.

3. Content (content)

  • techStack : Arrays of { name, version } for frontend & backend. Logos are auto-mapped.
  • projectGoal : A string supporting Markdown to explain the context of your work.
  • stats : Array of key metrics { label, value, description }.

Auto-mapped Icons & Contribution

TechCard automatically maps strings like "github" or "react" to their respective logos to keep your config clean. Is your favorite stack or social network missing? Add it!

How mapping works

// src/utils/icons.ts const SOCIAL_KEY_MAP: Record<string, string> = { "facebook": "fb", "fb": "fb", "linkedin": "in", "in": "in", "github": "github", "git": "github" }; /* Same for TECH_ICONS ("react", "laravel", etc.) */

How to contribute icons

  1. 1. Fork the repository & clone it locally.
  2. 2. Create a branch: git checkout -b feat/add-vue-icon
  3. 3. Add your image URL to src/utils/icons.ts in SOCIAL_ICONS or TECH_ICONS.
  4. 4. Push and open a Pull Request!
